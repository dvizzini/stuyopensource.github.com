pintweets.com is a hacker project I started last August. It uses Twitter's search and users/lookup API's, along with Google's Maps and Geocoding API's to map the location of Tweet's based on their Tweeter's stated location.

While the location's mapped are certainly not court-admissible evidence, it is a fun way to get a sense of who is tweeting what around the world.

Currently front-end only, I intend to improve PinTweets by refactoring the asynchronous API calls, implementing Twitter's Bootstrap for better cross-browser and mobile support, as well as Ruby on Rails to support OAuth and serve static pages. The repo for this project is here.

I started this refactoring process with Julian Cannon, whom I taught at a summer course at Berkeley, along with my coinstructor Blair Trump. At his web development firm, Blair implemented the algoirthm for a dance website.
